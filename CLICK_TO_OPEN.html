Coding Intro

What is coding?

Coding is just a list of instructions for a computer. Think of it like a recipe for a cake: the computer follows your steps exactly. Computers are powerful: They can solve massive equations like in a blink.

But they are not "smart": They don't think for themselves. If you forget a single comma or bracket, the computer will stop and say, "I don't understand".

Block vs. Text

Block Coding (like Scratch): You drag and drop pieces. It's great for beginners because it's impossible to make a "syntax error" (the pieces only fit if they are correct). Text Coding (like Python): This is what professionals use. It is faster and more powerful because it can use multiple CPU cores to handle heavy tasks. While it may be harder to learn, it gives unlimited "freedom" to build real apps, AI, and websites.

The Complete HTML Reference Guide

HTML stands for HyperText Markup Language. It is the special code used to build websites. Think of it like a set of building blocks. Each block, called a tag, tells the computer where to put text, pictures, or links.

The Basic Recipe

Every web page needs a starting structure. It is like the foundation of a house. Without it, the browser will not know how to show your work correctly.

First, we use <!DOCTYPE html>. This tells the computer that we are using modern HTML. It is a simple way to start every project. Don’t forget the “!” in front. And html is case-sensitive, at least for the most part.

Next, we use the <html> tag. This is the biggest container (a container is something that contains text, image, scripts, functions and more). Everything else you write must go inside the opening <html> and the closing </html>.

Something like this:

<!DOCTYPE html> Tells the browser what VERSION of html to use.

  <html> The biggest container, that contains all scripts, images, text and more.

    <head> Another container that's basically invisible but useful to store information.

    </head> The “closing” tag that tells the browser “This is the end of the <head> tag

</html> The “closing” tag that tells the browser “This is the end of the <html> tag

Inside the html tags, we have a <head>. This is the "brain" of the page. It holds hidden information. For example, the <title> tag goes here. Whatever you type in the title tag shows up on the browser tab at the top of your screen.

Finally, we have the <body>. This is the most important part for you. Everything you want people to see, like stories, photos, and buttons, must be placed inside the body tags.

Example of the simple HTML code:

<!DOCTYPE html> Tells the browser what VERSION of html to use.

<html> The biggest container, that contains all info, 

<head> The “brain” of the code that is invisible but needed.

    <title>My First Website</title> The title of the HTML page (your website).

</head> The end of the Head and tells the computer it ends.

<body> The start of the body (all info…)

    <p>Everything the user sees goes here!</p> A paragraph text.

</body> Ending of body.

</html> Ending of the whole thing.

 

Using Headings

Headings are like titles in a book. They make your page easy to read. HTML has six different sizes for headings.

The <h1> tag is the largest. It is usually used for the main title of your webpage. You should only use one <h1> per page.

The <h2> tag is for smaller titles, like chapter names. If you have even smaller sections, you can use <h3>, <h4>, <h5>, and even <h6>.

It is a good habit to use them in order. Do not jump from a big <h1> to a tiny <h5> just because you like the size. It is better to keep things organized.

Example of Headings:

<h1>The Great Space Adventure</h1> Big

<h2>Chapter 1: The Launch</h2> Smaller

<h3>Checking the launchpad</h3> Smaller

<h4>Check the Fuel</h4> Smaller

<h5>Safety Fifth</h5> Tiny

<h6>The Smallest Details</h6> Super Tiny

 

Writing Paragraphs and Text

The <p> tag is for normal text. Every time you want to start a new paragraph, you put your text inside <p> and </p>. The browser will automatically add a little space between paragraphs so they do not look too close together.

Sometimes you want to move to the next line without starting a whole new paragraph. For this, we use the <br> tag. It stands for "break." This tag is special because it does not need a closing tag.

If you want to draw a line across the page to separate two ideas, use the <hr> tag. This stands for "horizontal rule." It creates a thin grey line that helps keep your page looking neat.

Example of Paragraphs:

<p>This is my first paragraph about my Rubix’s cube.</p>

<p>This is my second paragraph.<br>I used a break to move to this line!</p>

<hr>

<p>Now the page is separated by a line.</p>

IMMPORTANT!!! ADDING A FONT FAMILY

Before this, we must learn to target. We will learn the targeting of overall tags in this lesson: Let's say we want to make the font of <body> the font ‘Arial’. Then we must first target body. We use <style> as CSS code. <style> lets you customize the appearance of your website, including fonts, backgrounds, outlines, highlights, etc. We start CSS like this:

<!DOCTYPE html>

<html>

<head>

    <title>Font Example</title>

    <style>

    </style>

</head>

<body>

    <p>

      Test area (paragraph)

    </p>

</body>

</html>

 

The <style> starts at line 5 and ends at line 6 with </style>

Now, the targeting part:

If we at brackets and a tag before it like this:

<style>body{}</style>

Here we have the body targeted, this would work, but it isn’t formatted properly.

We format it like this:

 

<style>

   body {

}

</style>

 

Anything inside the body tag is targeted with whatever styling is inside the ‘{}’

To add a font family now,  we use ‘font-family: ;’.

 

If we put them together we get:

<style>

   body {

font-family: Arial;

}

</style>

 

Now whatever is inside <body> is now Arial font: The full example looks like this:

 

<!DOCTYPE html>

<html>

<head>

    <title>Fonts Example</title>

    <style>

        body {

            font-family: Arial;

        }

    </style>

</head>

<body>

    <h1>This text will use the Arial font.</h1>

    <p>As well as this paragraph and all other text in the body.</p>

</body>

</html>

Making Text Look Weird, I mean Different

To make a word look very important, use the <strong> tag. This usually makes the text look bold and dark. It tells the reader. Basically bold, but bold is <b> which needs a closing tag after the bold text (</b>)

If you want to make text slanted, use the <em> tag. This stands for "emphasis." It makes the text look like italics which is the I symbol.

You can also use <u> to underline a word. This is helpful if you want to highlight a specific term, but be careful not to confuse it with a link!

Example of making text look different:

<p>I <strong>really</strong> like coding</p>

<p>The <em>wind</em>shifted through the trees.</p>

<p>Don't forget to <u>wash your hands</u> before <strong>EATING</strong> jeffery.</p>

 

Making Lists

Lists are great for organizing things. There are two main types of lists in HTML.

An Unordered List uses the <ul> tag. This creates a list with bullet points. Use this when the order doesn't make a difference, like a list of cool things.

An Ordered List uses the <ol> tag. This creates a list with numbers (1, 2, 3...). Use this when the order does matter, like the steps to make a peanut butter sandwich.

For both types of lists, you must put each item inside an <li> tag. This stands for "list item." You can have as many list items as you want!

Example of Lists:

<h3>Grocery List (Unordered)</h3>

<ul>

    <li>Apples</li>

    <li>Milk</li>

    <li>Bread</li>

</ul>

 

<h3>How to Make Toast (Ordered)</h3>

<ol>

    <li>Put bread in toaster.</li>

    <li>Press the lever down.</li>

    <li>Wait for it to pop up.</li>

</ol>

Adding Links and Pictures

The <a> tag is used to create a link. To tell the computer where the link goes, you add href="url" inside the tag. For example: <a href="https://www.nasa.gov">Visit Space!</a>.

The <img> tag is used to show a picture. It needs a "source" to know which picture to show. You write this as src="image-link".

Always add an alt="description" to your images. This is a short sentence describing the picture. If the picture doesn't load, or if someone cannot see well, the computer will read the "alt" text aloud.

Example of Links and Images:

<a href="[https://www.wikipedia.org](https://www.wikipedia.org)">Click here for facts!</a>

 

<img src="[https://picsum.photos/200](https://picsum.photos/200)" alt="A cool random picture">

 

Organizing Your Page

The <div> tag is like an empty box. You can put other tags inside it to keep them together. It is very helpful for moving large chunks of your website around later.

The <span> tag is a smaller box. It is used to grab just one or two words inside a paragraph so you can change their color or style without affecting the rest of the sentence.

The <header> tag is for the very top of your site, where the logo usually sits. The <footer> tag is for the very bottom, where you might put your name and the date.

Example of Organization:

<header>

    <h1>Welcome to My Blog</h1>

</header>

 

<div>

    <h2>Today's Post</h2>

    <p>I learned that <span style="color:blue">blue</span> is a cool color.</p>

</div>

 

<footer>

    <p>Created by Me in 2024</p>

</footer>

JavaScript — ABSOLUTE BASICS

This page only teaches the most basic JavaScript pieces and what they do.

Think of JavaScript like giving instructions to a website.

0. id — Naming an element (VERY IMPORTANT)

WHAT IT DOES:
Gives an HTML element a name so JavaScript can find it.

Example:

  1. <p id="title">Hello</p>

 

Here:

  • <p> is the paragraph
  • id="title" gives it the name title

Now JavaScript can target it.

Example JavaScript:

  1. document.getElementById("title")

 

This tells JavaScript:

"Find the element named title."

Another example:

  1. <input id="box">

 

Now JavaScript can read what the user typed from box.

Rule:

Every id must be unique (only one element can have that id).

1. <script> — Runs JavaScript

WHAT IT DOES:
Tells the browser that JavaScript code starts here.

Example:

  1. <script>
  2. alert("Hello");
  3. </script>

 

Without <script>, the browser will not run JavaScript.

2. alert() — Shows a popup message

WHAT IT DOES:
Creates a popup box on the screen.

Example:

  1. alert("Hello!");

 

Result:
A popup appears saying Hello!

3. let — Creates a variable

WHAT IT DOES:
Stores information so we can use it later.

Example:

  1. let name = "Alex";

 

Meaning:
We stored the word Alex inside something called name.

Think of it like a box that holds information.

4. function — A group of instructions

WHAT IT DOES:
Creates a set of instructions that can run later.

Example:

  1. function sayHello() {
  2.   alert("Hello!");
  3. }

 

Nothing happens yet.

The function runs only when we call it.

5. onclick — Runs code when something is clicked

WHAT IT DOES:
Runs JavaScript when a button is clicked.

Example:

  1. <button onclick="sayHello()">Click</button>

 

When the button is clicked, the sayHello function runs.

6. document.getElementById() — Finds something on the page

WHAT IT DOES:
Finds a specific HTML element using its id.

Example:

  1. document.getElementById("title")

 

This finds the element:

  1. <h1 id="title">Hello</h1>

 

7. .innerHTML — Changes text on the page

WHAT IT DOES:
Changes the text inside an HTML element.

Example:

  1. document.getElementById("title").innerHTML = "New Text";

 

Result:
The text becomes New Text.

8. .value — Gets text from an input box

WHAT IT DOES:
Reads what a user typed.

Example:

  1. let name = document.getElementById("box").value;

 

If the user typed Alex, then:

  1. name = "Alex"

 

9. if — Check if something is true

Explaining checkNumber(), (), and if in Functions

Let's look at this example:

  • <!DOCTYPE HTML>
  • <html>
  • <head>
  • <title>Simple JS Example</title>
  • </head>
  • <body>
  • <script>
  • function checkNumber() {
  •   let number = 7;
  •   if (number > 5) {
  •     alert("Number is bigger than 5!");
  •   }
  • }
  • checkNumber();
  • </script>
  • </body>
  • </html>

 

Step by Step Explanation:

  1. function checkNumber() { ... }
    • This defines a new function called checkNumber.
    • Think of it as writing a recipe. Nothing happens yet.
    • The { ... } part contains all the instructions that the function will run when called.
  2. let number = 7;
    • Inside the function, we create a variable named number and store the value 7.
    • Variables are like boxes that hold information we want to use later.
  3. if (number > 5) { ... }
    • This is a decision checker.
    • JavaScript asks: “Is the value of number greater than 5?”
    • If true, it runs the code inside { ... } (the alert).
    • If false, it does nothing.
  4. alert("Number is bigger than 5!");
    • If it is:
    • It shows a popup message on the screen.
  5. checkNumber();

THIS IS THE IMPORTANT PART, HERE WE ACTUALLY “USE” THE FUNCTION. TILL NOW, WE’VE JUST BEEN EXPLAINING WHAT THE FUNCTiON IS BUT NOT REALLY RUNNING THE FUNCTION

  • This is where we call the function.
  • Think of it as following the recipe we wrote earlier.
  • The code inside the function now runs: it checks the number and shows the alert if needed.

 Summary:

  • checkNumber() is the function name, () means “run it now.”
  • if checks a condition and only runs the code if the condition is true.
  • Everything inside { ... } is part of the instructions for the function or the if statement.
